# LINE Rich Menu 設計（Prototype v1）

日期：2026-06-23
狀態：設計定案，待建置

## 目標

為妙媽媽果園官方帳號建立聊天室底部的 Rich Menu，提供顧客最常用的三個入口。本版為 prototype，以最快上線為原則，採 **LINE Official Account Manager 後台手動設定**，不動到程式碼。

## 範圍

- 本版實作三格：**立即訂購、匯款回報、購買須知**。
- 未來擴充：**訂單查詢（B）** 待後端提供查詢 API 後再加入；屆時可能需重排版面或改用 Messaging API 程式化管理（見「未來考量」）。

## 實作方式

**方案：LINE OA Manager 手動設定（無程式碼）**

- 在官方帳號管理後台建立 Rich Menu、上傳圖片、框選區塊、設定各區動作。
- F 的自動回覆、C 的引導回覆，皆在 OA Manager 的「自動回覆訊息／關鍵字回覆」設定。
- 改動即時生效、零部署。
- 取捨：無法做「每位使用者不同選單」或「選單切換」等進階控制；prototype 階段不需要。

## 版面與圖片規格

- 尺寸：**精簡型 Compact，2500 × 843 px**
- 版型：**一排三欄**，每欄約 833 × 843 px
- 聊天室選單列文字（chat bar）：`選單`（或 `妙媽媽果園`）
- 預設狀態：開啟聊天室時自動展開

```
┌──────────────┬──────────────┬──────────────┐
│   立即訂購     │   匯款回報     │   購買須知     │
│     🍐        │     💰        │     📋        │
└──────────────┴──────────────┴──────────────┘
   區域 A           區域 C          區域 F
```

圖片設計建議（需另製圖，可用 Canva 等工具）：
- 沿用訂單卡片品牌色：主橘 `#E89B3C`、文字棕 `#6B4E32`、底色奶油 `#FBF3E0`、輔助綠 `#3E8E41`
- 三欄各一個圖示 + 中文標籤，欄與欄之間以細分隔線或留白區隔
- 標籤文字：`立即訂購` ／ `匯款回報` ／ `購買須知`

## 各區塊行為

### 區域 A — 立即訂購
- 動作類型：**連結（URI）**
- 連結網址：`https://liff.line.me/{LIFF_ID}`
  - 在 LINE 內開啟 LIFF 商店頁（前端 `/` 路由）讓顧客直接下單。
  - `{LIFF_ID}` 填入專案實際的 LIFF App ID（與前端 `VITE_MIAO_LIFF_ID` 相同）。

### 區域 C — 匯款回報
- 動作類型：**送出文字**
- 送出內容（關鍵字）：`我要回報匯款`
- 重要前提：Rich Menu 的「送出文字」是**點擊即刻送出**，無法讓使用者先填空。因此採「送關鍵字 → 自動回覆引導格式」模式。
- 對應自動回覆（關鍵字 `我要回報匯款`）：
  ```
  感謝您的訂購 🍐
  請依下列格式回覆，確認款項後盡快為您安排出貨：

  訂單編號：MM-______
  帳號末5碼：______
  匯款金額：______
  ```

### 區域 F — 購買須知
- 動作類型：**送出文字**
- 送出內容（關鍵字）：`購買須知`
- 對應自動回覆（關鍵字 `購買須知`）：
  ```
  🍐 妙媽媽果園 購買須知

  ・運費 NT$150，單筆滿 NT$5,000 免運
  ・付款方式：轉帳匯款，請於下單後 3 日內完成
  ・匯款後請點選「匯款回報」告知帳號末5碼與金額
  ・確認款項後安排出貨

  有任何問題歡迎直接傳訊息給我們 😊
  ```
  - 運費規則來源：後端 `app/core/constants.py`（`SHIPPING_FEE=150`、`FREE_SHIPPING_THRESHOLD=5000`）。日後若調整門檻，須同步更新此自動回覆文字。

## 相依與注意事項

- 需要官方帳號具備 Rich Menu 與自動回覆功能（一般 LINE OA 皆有）。
- A 區依賴 LIFF 已正確設定且 Endpoint 指向 Vercel production 網址。
- C、F 的自動回覆與 Rich Menu 是兩處設定，需一起完成才會運作。
- 文案內含的運費/門檻為「寫死的文字」，與後端常數無自動連動，調價時要記得手動同步。
