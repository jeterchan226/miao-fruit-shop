# 設計：店內平板後台「新訂單即時提醒」

日期：2026-06-26

## 背景與目標

下單成立後，後端已 push 訂單明細給**客戶**（`line_service.send_order_created`）。但店家端無法即時得知有人下單：LINE 官方帳號的「聊天」列表排序與通知由使用者主動傳入的訊息驅動，OA 主動 push 出去的訊息不會頂上列表、也不會通知小編；加上訂單走 LIFF 而非聊天訊息，客戶若沒傳訊息給 OA，店家就不會自動感知到訂單。

評估過幾條通知管道（LINE 群組 push、Email、出單印表機）後，決定先做**店內一台平板固定開著訂單後台**的即時提醒：有新訂單就響鈴 + 視覺提示。理由：零硬體、零月費、通知可靠，最快上線；量做大後再評估出單印表機。

目標：平板開著訂單後台時，新訂單即時以**聲音 + 畫面提示**通知店家，並把未處理訂單導引到眼前。

## 決策

- **觸發範圍**：僅「後台開著（前景、螢幕亮）」時負責提醒。平板鎖屏 / 瀏覽器切到背景的通知**不在範圍內**（那需 Web Push + Service Worker + VAPID + 訂閱存 DB + 後端推播，工程明顯更重，YAGNI）。
- **機制**：純前端輪詢既有 API，**後端零改動**。前端每 ~20 秒輪詢 `GET /api/admin/orders/summary`，比對 `total_orders`（只增不減，天然的新訂單訊號）。增加即代表有新單。
- **不做**：Web Push / Service Worker / PWA 安裝、背景通知、LINE 群組推播、出單印表機。皆延後或不採用。

## 範圍

僅前端（`frontend/src/AdminApp.jsx` 及相關）。後端、資料庫、既有客戶通知流程皆不變。

---

## 變更點（前端）

### 1. 輪詢與新訂單偵測

在 AdminApp 已登入的外層（不限於訂單分頁）加入輪詢：

- 以 `setInterval` 每 ~20 秒呼叫既有 `getAdminOrderSummary`（對應 `GET /api/admin/orders/summary`，回傳 `total_orders`）。
- 保存「上次看到的 `total_orders`」於 ref / state。
- 新值 > 舊值 → 判定有 `delta = new - old` 筆新訂單 → 觸發提醒，並刷新目前訂單列表與 `status_counts`。
- 首次載入只記錄基準值，不觸發提醒（避免一進後台就誤報）。
- 元件卸載 / 登出時清除 interval。

### 2. 提醒行為

- **聲音**：播放短提示音。
  - 瀏覽器自動播放政策會擋未經使用者手勢的音訊。解法：在登入後第一個使用者手勢（登入按鈕點擊，或一個明確的「啟用提醒音 🔔」開關）時先解鎖（預載並以靜音 play() 解鎖 `Audio` 物件）。
  - 提供開關讓店家可開 / 關提示音，狀態存 `localStorage`。
- **視覺**：畫面上方顯示橫幅 / toast「🔔 有 N 筆新訂單」。
  - 點擊 → 切到訂單分頁並套用待付款（`pending_payment`）篩選，重新載入列表。
  - 訂單分頁導覽的紅點 / 數字同步反映待處理數。
  - 橫幅在點擊或手動關閉前持續顯示。

### 3. 防螢幕睡眠（Wake Lock）

- 後台處於前景時，呼叫 `navigator.wakeLock.request('screen')` 讓平板螢幕不自動睡。
- 監聽 `visibilitychange`，回到前景時重新取得 wake lock（鎖會在切背景時自動釋放）。
- 不支援 Wake Lock API 的裝置 → try/catch 略過，不影響其他功能（graceful degradation）。

### 4. 放置位置

輪詢、提醒、Wake Lock 邏輯放在 AdminApp 已驗證後的外層元件 / hook，使其**不論當下在訂單或商品分頁都會運作**。

## 測試（前端）

1. **偵測新訂單**：模擬輪詢回傳的 `total_orders` 從 N → N+2 → 觸發提醒、`delta` 為 2。
2. **不誤報**：`total_orders` 不變 → 不觸發提醒。
3. **首次不誤報**：首次輪詢只設基準值，不觸發。
4. **音訊解鎖**：未解鎖前不嘗試播放；解鎖後新訂單會呼叫播放。
5. **提示音開關**：關閉時不播放、僅顯示視覺提示；開關狀態持久化於 `localStorage`。

（使用假計時器 / mock fetch 進行。）

---

## 不變更項目

- 後端、資料庫 schema、訂單建立的金額 / 庫存 / 既有客戶通知流程不動。
- 不新增 Web Push / Service Worker / PWA / 背景通知。
- 不新增 LINE 群組推播或出單印表機整合（日後量大再評估）。
